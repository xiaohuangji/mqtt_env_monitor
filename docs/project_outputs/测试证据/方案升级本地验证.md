# MQTT 方案升级本地验证（MQTT 5.0 + 多 payload 格式）

## 1. 验证目的

验证 Issue #25 的客户端升级：节点与监控端在 MQTT 5.0（含 Topic Alias）与 MQTT 3.1.1 兼容模式下均可收发，三种 payload 格式（json / json-min / msgpack）端到端解析正确。

## 2. 验证环境

| 项 | 值 |
| --- | --- |
| 日期 | 2026-06-11 |
| Broker | Mosquitto 2.1.2（本地台架，localhost:1883，MQTT 5.0，topic_alias_max=10） |
| 客户端库 | paho-mqtt 2.1.0（Python 3.11） |
| 监控端 | `web_monitor.py --mqtt-version 5 --web-port 8090` |

> EMQX 完整链路验证在云服务器开机后补做（同一套命令把 `--host` 指向服务器即可），不影响本次客户端行为验证。

## 3. 验证方法

监控端（v5）订阅 `env_monitor/+/+`，并发运行 4 个节点进程覆盖组合矩阵，每个节点发 3 轮：

| 节点 | 协议 | 格式 | QoS | Topic Alias |
| --- | --- | --- | --- | --- |
| node01 | 5.0 | json | 1 | on（默认） |
| node02 | 5.0 | json-min | 1 | on |
| node03 | 5.0 | msgpack | 1 | on |
| node04 | 3.1.1 | json | 2 | —（3.1.1 无此特性） |

## 4. 验证结果

`/api/data` 快照：**received_total=15，invalid_total=0**，格式计数 `{"json": 9, "json-min": 3, "msgpack": 3}`，四个节点的数据全部正确显示。

### 4.1 Topic Alias 生效（node01 日志节选）

```text
connected to 127.0.0.1:1883 as sim_node01 topic_alias_max=10
publish ok topic=env_monitor/node01/temperature qos=1 alias=1(announce) payload={...}
publish ok topic=env_monitor/node01/humidity    qos=1 alias=2(announce) payload={...}
publish ok topic=env_monitor/node01/temperature qos=1 alias=1 payload={...}   ← 空主题发布
publish ok topic=env_monitor/node01/humidity    qos=1 alias=2 payload={...}   ← 空主题发布
```

连接内首条消息携带完整主题注册别名，其后消息以空主题 + 别名发布（每条节省"主题长度"字节数，量化数据由抓包实验给出）；监控端收到的消息主题由 Broker 还原，显示无影响。

### 4.2 紧凑格式归一化

- node02（json-min）：payload 无 unit 字段，页面正确显示 `lux`（监控端按数据类型反查）；epoch 毫秒时间戳还原为 ISO 格式显示。
- node03（msgpack）：二进制 payload 解码正确，`noise → dB` 反查正确。

### 4.3 兼容模式

node04 以 MQTT 3.1.1 + QoS 2 正常连接、发布，监控端（v5 客户端）正常接收——新旧协议客户端可在同一 Broker 上混跑。

## 5. 结论

- MQTT 5.0 升级（Topic Alias、会话过期参数）与三种 payload 格式全部工作正常，3.1.1 兼容模式保留可用。
- 字段映射与《网络负载与成本建模.md》第 4.3 节规格一致。
- 遗留事项：EMQX 服务器端到端复验（服务器下次开机时执行，预计 5 分钟）。

## 6. 复现命令

```bash
# 1. 本地 Broker（Mosquitto 安装后）
& "C:\Program Files\mosquitto\mosquitto.exe" -v

# 2. 监控端（MQTT 5.0）
D:\anaconda\python.exe src\monitor\web_monitor.py --web-port 8090

# 3. 各组合节点（另开终端）
D:\anaconda\python.exe src\simulated_nodes\sim_node.py --node-id node01 --types temperature,humidity --interval 1 --count 3 --qos 1
D:\anaconda\python.exe src\simulated_nodes\sim_node.py --node-id node02 --format json-min --types light --interval 1 --count 3 --qos 1
D:\anaconda\python.exe src\simulated_nodes\sim_node.py --node-id node03 --format msgpack --types noise --interval 1 --count 3 --qos 1
D:\anaconda\python.exe src\simulated_nodes\sim_node.py --node-id node04 --mqtt-version 311 --types temperature --interval 1 --count 3 --qos 2

# 4. 浏览器打开 http://127.0.0.1:8090/ 或 curl http://127.0.0.1:8090/api/data
```
