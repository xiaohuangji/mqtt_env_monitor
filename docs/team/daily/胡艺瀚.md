3. 遇到的问题

今天主要遇到的问题有三个：

- 项目初期任务较多，如果一次性规划过大，容易导致后续执行压力过大。
- 小组成员分工需要既符合课程要求，又要兼顾软硬件结合的实际实现难度。
- 在设计 MQTT Topic 和 JSON 字段时，需要判断是否可以使用中文命名。

此外，通过老师对工程思维的强调，我意识到如果只完成代码和界面，可能无法充分体现课程设计要求。项目还需要从需求是否清楚、方案是否合理、测试是否有数据、过程是否可追溯等角度进行规划。

4. 解决方法

针对项目推进问题，最终确定采用“一个模块一个模块推进”的流程，不一次性创建过多 Issue，而是每次只围绕当前模块完成设计、实现、提交和关闭。

针对小组分工问题，根据项目技术路线和成员任务需要，将工作划分为软件侧和硬件侧：本人和曾家豪负责软件侧，孙博宇和韦煜城负责硬件侧。这样既能保证软件原型先行，又能为后续 ESP32 硬件节点接入预留空间。

针对命名问题，最终确定文档和页面展示可以使用中文，但 MQTT Topic、JSON 字段、代码变量和配置项统一使用英文，以减少编码兼容、抓包分析和跨语言开发中的问题。

针对工程思维体现不足的问题，决定在项目中通过 GitHub Issue、commit、README 进度记录、会议记录、需求文档、协议设计文档和后续测试证据来保留完整过程，使项目不仅能运行，还能说明设计依据、验证方式和改进方向。
