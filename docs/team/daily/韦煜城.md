在硬件接口方面，学习了 ESP32 丰富的 GPIO 接口资源，以及 GPIO、ADC、I2C、单总线等不同接口方式的基本原理和适用场景。其中，普通 GPIO 可用于数字输入输出；ADC 可用于采集模拟电压信号，适合噪声、气敏、土壤湿度等模拟量传感器；I2C 适合连接 BH1750 光照传感器等总线设备；单总线通信常用于 DHT 系列温湿度传感器。通过对接口类型的梳理，为后续传感器选型和接线设计提供了依据。
在系统稳定性方面，初步了解了 ESP32 的断线重连、串口日志调试和低功耗运行特点。ESP32 在物联网项目中通常需要长期运行，因此后续程序中需要考虑 Wi-Fi 连接状态检测、MQTT 断线重连、上传周期控制和串口调试输出等功能，以提高系统运行稳定性和可维护性。

4.解决方法
针对 Wi-Fi 模式理解不清的问题，通过查阅资料和对比示例代码进行学习。最终明确 Station 模式是 ESP32 作为客户端连接路由器，本项目后续连接 Wi-Fi 和 MQTT Broker 时主要采用该模式；AP 模式则是 ESP32 自身创建热点，供手机或电脑连接，适合设备配网或局部控制场景。通过串口查看 IP 地址、连接状态等信息，进一步理解了 ESP32 接入网络的过程。
针对接口类型混淆的问题，采用逐个拆解的方式进行学习。将 GPIO 理解为通用数字输入输出接口；将 ADC 理解为模拟电压采集接口；将 I2C 理解为两线同步通信总线，可挂接多个设备；将单总线理解为只使用一个数据引脚进行通信的专用协议。结合后续可能使用的 DHT22、BH1750、噪声传感器等模块，分别对应其接口类型，从而加深理解。
针对 ESP32 低功耗和系统稳定性问题，通过阅读芯片架构简介和物联网项目案例，初步理解了低功耗模式适合电池供电、周期性采集和远程部署等场景。虽然本课程设计阶段主要以功能实现和串口调试为主，但后续仍可在报告中说明低功耗设计的扩展价值。
同时，通过 AI 工具辅助解释专业概念，并结合课程设计实际需求进行整理，逐步将零散知识转化为与本项目相关的设计思路。
