#### 草稿记录

1. 需求分析：讨论后，初步得出对温度、湿度、光照和噪声等的检测，可能在天气预报/监控、智慧农业（农业大棚）、智能家居（控制室内温/湿度）、噪声检测等方面进行广泛应用。
2. 技术路线确立：软硬件结合，先用软件模拟节点，用本地 MQTT Broker 接收并转发，在 Web 监控端订阅主题并显示数据，后续换成硬件节点。
3. 可能遇到的问题：接口不统一；量化指标不好统计。
4. 分工：胡艺瀚、曾家豪负责软件侧；孙博宇、韦煜城负责硬件侧。
5. 工作流程讨论：GitHub 维护项目，项目成员应积极提交 PR 和 Issue；每日晚 22:00 ~ 24:00 应在项目群内分享当天成果与进度；每两日进行一次会议。
6. 近期任务：建立项目仓库、熟悉工作流程，对项目相关的计网知识和 Git/GitHub 操作进行详细学习，规划自己负责部分的具体实现方案。
7. 产品性能追求：追求在准确性、实时性、可靠性上达到较好的水平，最好能设计实验进行测试，得到量化的评价指标。

#### 正式总结

1. 需求分析：本项目面向多节点环境监测场景，初步聚焦温度、湿度、光照和噪声等环境数据。相关应用场景包括天气监控、智慧农业大棚、智能家居环境调节和噪声检测等。
2. 技术路线：项目采用软硬件结合方案。第一阶段使用软件模拟环境监测节点，通过本地 MQTT Broker 接收并转发数据，由 Web 监控端订阅主题并显示数据；后续阶段逐步替换为 ESP32 等真实硬件节点。
3. 可能问题：后续实现中需要重点关注软硬件接口统一问题，以及准确性、实时性、可靠性等量化指标的统计方法。
4. 小组分工：胡艺瀚、曾家豪负责软件侧工作；孙博宇、韦煜城负责硬件侧工作。
5. 工作流程：项目使用 GitHub 进行维护，小组成员通过 Issue 和 Pull Request 记录任务、问题和修改过程；每日 22:00 ~ 24:00 在项目群内同步当天成果与进度；每两日召开一次小组会议。
6. 近期任务：完成项目仓库建立，熟悉 GitHub 协作流程，学习 MQTT、网络通信、ESP32、测试抓包等相关知识，并规划各自负责部分的具体实现方案。
7. 性能目标：系统后续应围绕准确性、实时性和可靠性设计测试实验，并尽量形成消息到达率、平均延迟、异常情况统计等可量化评价指标。

#### 形成决议

1. 确认项目采用“软件模拟节点 + 本地 MQTT Broker + Web 监控端 + 后续硬件节点接入”的技术路线。
2. 确认软件侧由胡艺瀚、曾家豪负责，硬件侧由孙博宇、韦煜城负责。
3. 确认使用 GitHub 维护项目，并通过 Issue、PR、会议记录和每日同步保留项目过程材料。
4. 确认每两日召开一次小组会议，每日 22:00 ~ 24:00 在项目群内同步进度。
5. 确认后续测试需要关注准确性、实时性、可靠性和量化评价指标。
