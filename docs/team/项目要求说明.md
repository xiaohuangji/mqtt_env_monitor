# 项目要求说明

## 项目目标

完成一个基于 MQTT 的多节点环境监测系统，体现数据通信与网络技术课程中的协议设计、网络分层、数据传输、测试验证和工程分析能力。

## 小组信息

- 小组成员：胡艺瀚、孙博宇、韦煜城、曾家豪
- 指导老师：和洁、傅攀峰、刘若辰

## 阶段任务

1. 题目理解与任务分工：明确课程要求、系统目标和成员职责。
2. 需求分析与技术比较：分析应用场景、功能需求、性能指标，比较 MQTT 与 HTTP。
3. 总体方案设计：设计系统结构、模块划分、数据流向、主题结构和消息格式。
4. 核心原型实现：完成 MQTT Broker、模拟节点和监控端原型。
5. 系统联调：完成多节点同时上传、监控端订阅显示和基础异常处理。
6. 测试验证：完成 QoS、延迟、到达率、断线重连等测试。
7. 文档整理：完善报告、PPT、日志、测试附件和演示材料。

## 代码维护要求

- PC 模拟节点代码放在 `src/simulated_nodes/`。
- MQTT Broker 配置或说明放在 `src/broker/`。
- 监控端代码放在 `src/monitor/`。
- 后端服务代码放在 `src/server/`。
- 公共工具、消息格式和常量放在 `src/common/`。
- ESP32 硬件节点代码放在 `hardware/esp32_nodes/`。
- 硬件连接图、接线说明和传感器资料放在 `hardware/wiring/`。

## 测试维护要求

- Wireshark 抓包文件放在 `captures/wireshark/`。
