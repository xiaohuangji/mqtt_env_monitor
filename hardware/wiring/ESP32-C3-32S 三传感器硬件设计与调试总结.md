# ESP32-C3-32S 三传感器硬件设计与调试总结

## 1. 文档说明

本文档用于整理“基于 MQTT 协议的环境监测系统”中 ESP32 硬件节点的硬件设计与调试过程。内容整合了前期的 **ESP32-C3-32S 三传感器硬件设计总结** 和 **ESP32-C3-32S 硬件调试过程问题及解决方案总结**，形成一份完整的硬件设计说明文档。

本阶段硬件节点主要完成以下工作：

- 确认 ESP32 开发板型号；

- 查阅并确认传感器模块接口含义；

- 完成 MQ-2、MH-RD、FC-28 三个传感器的引脚分配；

- 完成 ESP32-C3-32S 与各传感器的连接设计；

- 编写并调试三传感器采集程序；

- 接入 WiFi 和 NTP 时间同步；

- 在串口监视器中输出符合项目 MQTT JSON 格式的数据；

- 总结硬件调试过程中出现的问题及解决方案。

本次任务只完成传感器采集和串口输出检测，暂不进行 MQTT 实际发布，后续可在现有代码基础上继续接入 MQTT Broker。

## 2. ESP32 开发板型号说明

本项目当前使用的主控板型号为 **ESP32-C3-32S**，也可称为 **ESP-C3-32S**。该开发板属于 ESP32-C3 系列，与普通 ESP32-WROOM 系列在核心架构、GPIO 数量和外设资源上存在差异。

### 2.1 ESP32-C3-32S 基本特点

| 项目 | 说明 |

| --- | --- |

| 芯片系列 | ESP32-C3 |

| CPU 架构 | RISC-V 单核 32 位 |

| 主频 | 最高约 160 MHz |

| 无线功能 | 2.4 GHz WiFi，支持 BLE 5.0 |

| ADC | 支持多路 ADC 输入，适合读取模拟传感器 |

| 开发环境 | VS Code + PlatformIO + Arduino Framework |

| 串口特点 | 支持 USB CDC，需要在 PlatformIO 中正确配置 |

ESP32-C3-32S 适合作为低功耗物联网节点，用于采集环境数据并通过 WiFi 进行数据传输。本项目中，ESP32-C3-32S 作为硬件采集节点，负责读取传感器模拟量，并将采集结果封装成符合项目协议的 JSON 数据。

### 2.2 与普通 ESP32 的区别

在调试过程中发现，普通 ESP32 常用的 GPIO32、GPIO33、GPIO34、GPIO35 等引脚并不适用于 ESP32-C3-32S。因此不能直接照搬普通 ESP32 的引脚分配方案，必须根据 ESP32-C3-32S 的实际可用 ADC 引脚重新设计。

本项目最终选择使用 GPIO0、GPIO1、GPIO2 作为三个传感器的模拟量输入引脚。

## 3. 本阶段接入的传感器说明

本次硬件节点只接入三个传感器模块：

1. MQ-2 气敏传感器模块；

2. MH-RD 雨滴传感器模块；

3. FC-28 土壤湿度传感器模块。

DHT22、BH1750、声音传感器等其他模块暂未接入，后续拿到硬件或需要扩展时再按照相同模块化结构继续增加。

### 3.1 传感器功能说明

| 传感器模块 | 对应数据类型 | 功能说明 | 本次使用接口 |

| --- | --- | --- | --- |

| MQ-2 气敏传感器 | `gas` | 用于检测环境中烟雾、可燃气体等气体浓度变化 | AO |

| MH-RD 雨滴传感器 | `rain` | 用于检测雨滴或检测板表面湿润程度 | AO |

| FC-28 土壤湿度传感器 | `soil_moisture` | 用于检测土壤含水量变化 | AO |

三个传感器模块都带有模拟输出 AO 和数字输出 DO。为了便于观察连续变化，本阶段优先使用 AO 模拟输出进行 ADC 采集，DO 数字输出暂不接入。

## 4. 各传感器模块接口功能确认

在接线前，需要先确认每个传感器模块上 VCC、GND、AO、DO 等接口含义。三个模块虽然功能不同，但常见接口结构基本一致。

### 4.1 通用接口说明

| 接口 | 名称 | 功能说明 |

| --- | --- | --- |

| VCC | 电源正极 | 给传感器模块供电，本项目接 ESP32-C3-32S 的 3.3V |

| GND | 电源地 | 接 ESP32-C3-32S 的 GND，保证共地 |

| AO | Analog Output，模拟输出 | 输出随环境变化而变化的模拟电压，接 ESP32-C3 ADC 引脚 |

| DO | Digital Output，数字输出 | 由模块上的 LM393 比较器和电位器阈值决定，输出高/低电平 |

### 4.2 MQ-2 气敏传感器接口说明

| 引脚 | 功能 |

| --- | --- |

| VCC | 模块供电引脚，接 3.3V |

| GND | 地线，接 ESP32-C3 GND |

| AO | 气体浓度相关的模拟电压输出，接 ESP32-C3 ADC |

| DO | 当气体浓度超过设定阈值时输出数字电平，本次暂不接 |

MQ-2 模块上通常带有电位器，可调节 DO 输出阈值。本项目主要读取 AO 原始 ADC 值，观察气体环境变化趋势，因此暂不使用 DO。

### 4.3 MH-RD 雨滴传感器接口说明

| 引脚 | 功能 |

| --- | --- |

| VCC | 模块供电引脚，接 3.3V |

| GND | 地线，接 ESP32-C3 GND |

| AO | 雨滴检测板湿润程度对应的模拟电压输出 |

| DO | 当雨滴或湿润程度达到阈值时输出数字电平，本次暂不接 |

雨滴传感器一般由检测极板和 LM393 主控板组成。检测板越湿，AO 输出值通常会发生明显变化，可用于判断雨滴状态或湿润程度。

### 4.4 FC-28 土壤湿度传感器接口说明

| 引脚 | 功能 |

| --- | --- |

| VCC | 模块供电引脚，接 3.3V |

| GND | 地线，接 ESP32-C3 GND |

| AO | 土壤湿度相关的模拟电压输出 |

| DO | 当土壤湿度达到设定阈值时输出数字电平，本次暂不接 |

FC-28 由双针探头和 LM393 主控板组成。一般情况下，土壤越湿，模拟输出对应的 ADC 读数会发生变化。本项目先读取 ADC 原始值，后续可根据实测数据进行标定和换算。

## 5. 引脚分配设计

### 5.1 引脚分配原则

由于 ESP32-C3-32S 与普通 ESP32 的引脚资源不同，本项目重新规划传感器接入引脚，遵循以下原则：

1. 优先使用 ESP32-C3-32S 支持 ADC 的 GPIO；

2. 本阶段只使用 AO 模拟输出，不占用额外 DO 引脚；

3. 避免使用普通 ESP32 中才存在的 GPIO32、GPIO33、GPIO35；

4. 所有传感器统一使用 3.3V 供电，降低 ADC 输入超压风险；

5. 引脚定义统一写入 `config.h`，便于后续修改和扩展。

### 5.2 最终引脚分配表

| 传感器模块 | 数据类型 | 传感器接口 | ESP32-C3-32S 引脚 | 说明 |

| --- | --- | --- | --- | --- |

| MQ-2 气敏传感器 | `gas` | AO | GPIO0 | 读取气敏传感器模拟量 |

| MH-RD 雨滴传感器 | `rain` | AO | GPIO1 | 读取雨滴传感器模拟量 |

| FC-28 土壤湿度传感器 | `soil_moisture` | AO | GPIO2 | 读取土壤湿度传感器模拟量 |

### 5.3 电源与地线分配

| 传感器模块 | VCC | GND |

| --- | --- | --- |

| MQ-2 气敏传感器 | ESP32-C3 3.3V | ESP32-C3 GND |

| MH-RD 雨滴传感器 | ESP32-C3 3.3V | ESP32-C3 GND |

| FC-28 土壤湿度传感器 | ESP32-C3 3.3V | ESP32-C3 GND |

所有传感器必须与 ESP32-C3 共地，否则 ADC 采集值可能异常或不稳定。

## 6. ESP32-C3-32S 与各传感器连接描述

### 6.1 MQ-2 气敏传感器连接

MQ-2 气敏传感器用于检测环境中气体浓度变化。本项目中只使用其 AO 模拟输出。

连接方式如下：

| MQ-2 引脚 | ESP32-C3-32S 引脚 |

| --- | --- |

| VCC | 3.3V |

| GND | GND |

| AO | GPIO0 |

| DO | 暂不接 |

ESP32-C3 通过 GPIO0 读取 MQ-2 的 ADC 原始值，并将其作为 `gas` 类型数据输出。

### 6.2 MH-RD 雨滴传感器连接

MH-RD 雨滴传感器用于检测雨滴或板面湿润程度。本项目中只使用 AO 模拟输出。

连接方式如下：

| MH-RD 引脚 | ESP32-C3-32S 引脚 |

| --- | --- |

| VCC | 3.3V |

| GND | GND |

| AO | GPIO1 |

| DO | 暂不接 |

ESP32-C3 通过 GPIO1 读取雨滴传感器 ADC 原始值，并将其作为 `rain` 类型数据输出。

### 6.3 FC-28 土壤湿度传感器连接

FC-28 土壤湿度传感器用于检测土壤湿度。本项目中只使用 AO 模拟输出。

连接方式如下：

| FC-28 引脚 | ESP32-C3-32S 引脚 |

| --- | --- |

| VCC | 3.3V |

| GND | GND |

| AO | GPIO2 |

| DO | 暂不接 |

ESP32-C3 通过 GPIO2 读取土壤湿度传感器 ADC 原始值，并将其作为 `soil_moisture` 类型数据输出。

## 8. 硬件调试过程问题及解决方案

### 8.1 ESP32 型号与原引脚设计不匹配

#### 问题现象

初期参考普通 ESP32 的引脚规划时，曾将三个传感器 AO 分配到 GPIO32、GPIO33、GPIO35。后来确认当前开发板型号为 ESP32-C3-32S，该型号并不具备这些引脚。

#### 原因分析

ESP32-C3 与普通 ESP32-WROOM 系列不同，GPIO 资源较少，不能直接使用普通 ESP32 的高编号 ADC 引脚。

#### 解决方案

重新规划 ESP32-C3-32S 的 ADC 引脚：

| 传感器 | 原错误引脚 | 修改后引脚 |

| --- | --- | --- |

| MQ-2 | GPIO32 | GPIO0 |

| MH-RD | GPIO33 | GPIO1 |

| FC-28 | GPIO35 | GPIO2 |

并在 `config.h` 中统一维护引脚宏定义。

---

### 8.2 ESP32-C3 的串口无显示

#### 问题现象

ESP32-C3 的串口无显示

#### 出现问题原因

ESP32-C3 的引脚与各模块接线存在错误

#### 解决方案及后续操作说明

在正确接线及系统初始化后，串口输出能够正常显示。依次对土壤湿度传感器（FC-28）和雨滴传感器（MH-RD）进行功能验证。

土壤湿度传感器校准与测量方法

ADC 最大值校准：在空气干燥条件下，采集传感器的模拟输出信号，并将其 ADC 值定义为 ADC_max = 4095。

ADC 最小值校准：将传感器浸入水中，在饱和条件下采集 ADC 信号，进行 10 次采样并取平均值，得到 ADC_min。

雨滴传感器同样采用模拟输出 AO 进行测量。

干燥板状态：在无水条件下采集 AO 模拟信号，定义最大值 ADC_max。

湿润状态：在板面湿润条件下采集 AO 信号，取多次平均值作为 ADC_min。
